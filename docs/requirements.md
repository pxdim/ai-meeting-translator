# 需求文件

## 使用者描述

一位企業主需要一套個人專用的網頁系統，用於會議記錄。

## 功能需求

### 核心功能（MVP）

1. **即時語音辨識**
   - 支援中英文混合語音（約 50/50 比例）
   - 顯示逐字稿
   - 信賴度指標

2. **即時翻譯**
   - 雙語並排顯示（中文 | 英文）
   - 即時更新
   - 流暢的捲動體驗

3. **音訊錄製**
   - 瀏覽器原生錄音
   - 完整會議儲存
   - 可重播

4. **會議管理**
   - 會議歷史列表
   - 會議詳情檢視
   - 編輯會議標題

5. **匯出功能**
   - PDF 格式
   - DOCX 格式
   - TXT 純文字

### 進階功能（第二階段）

1. **AI 會議筆記**
   - 自動摘要
   - 行動項目擷取
   - 關鍵決策記錄

2. **搜尋功能**
   - 關鍵字搜尋
   - 跨會議搜尋

3. **音訊同步**
   - 點擊逐字稿跳轉對應音訊位置
   - 音訊播放時高亮對應文字

## 非功能需求

### 使用者體驗

- **快速啟動**: < 3 秒從開啟到開始錄音
- **簡潔介面**: 一鍵開始錄音
- **即時回饋**: 錄音狀態、連線狀態清楚顯示

### 技術限制

- **跨瀏覽器支援**: Chrome、Safari、Firefox
- **雲端託管**: 可部署到 VPS
- **單一使用者**: 不需要登入系統
- **資料隱私**: API 金鑰不暴露在前端

### 可靠性

- **穩定錄音**: 支援 1 小時以上連續錄音
- **資料安全**: 音訊和逐字稿正確儲存
- **錯誤處理**: API 失敗時有適當提示

## 使用場景

### 1. 內部會議

團隊定期會議，需要記錄討論內容和決策。

### 2. 客戶會議

與客戶的會議記錄，確保需求正確記錄。

### 3. 場刊/會議

大型活動或會議的記錄，需要準確的逐字稿。

### 4. 客戶需求討論

記錄客戶提出的具體需求和變更。

## 成本考量

- 接受付費 API
- 預估每小時會議成本 < $1
- 每月 20 小時約 $15 API 成本
